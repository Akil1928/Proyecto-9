Informe Técnico - Sprint 1
Sistema de Gestión Académica

1. Introducción

El Sprint 1 tiene como objetivo configurar el proyecto Maven con JavaFX, implementar las variantes de listas enlazadas solicitadas y construir el módulo de Expediente Académico. Este módulo permite registrar cursos cursados por un estudiante, mostrar su información en orden cronológico, eliminar cursos y navegar dentro del historial de vistas usando una lista circular doble.

2. Arquitectura del sistema

El proyecto sigue el patrón MVC. La capa de modelo contiene las entidades, servicios y estructuras de datos. La capa de vista contiene los archivos FXML y CSS. La capa de controlador se encarga de conectar la interfaz con la lógica del sistema.

La estructura del proyecto se organizó bajo el paquete cr.ac.ucr.sga, siguiendo la jerarquía solicitada para Maven. La clase Main inicia la aplicación JavaFX, ViewFactory centraliza la carga de vistas, MainController controla el menú principal y StudentController controla el expediente académico.

3. Patrones de diseño utilizados

MVC:
Se utiliza para separar la lógica de negocio, los datos y la interfaz gráfica. Los controladores reciben eventos desde JavaFX, llaman servicios del modelo y actualizan los controles visuales.

Factory Method:
La clase ViewFactory centraliza la creación y carga de vistas FXML. Esto evita que la clase Main tenga que conocer detalles internos de la carga de pantallas.

Singleton:
AcademicRecordService y SessionHistoryService usan Singleton para garantizar una única instancia compartida durante toda la ejecución.

Builder:
StudentBuilder y CourseBuilder permiten construir objetos Student y Course paso a paso con validaciones antes de crear la instancia final.

4. Estructuras de datos implementadas

Las cuatro listas fueron portadas y adaptadas desde el proyecto PG-04 del curso, reutilizando la lógica ya probada y ajustando los nombres de métodos y el uso de nodos con getters/setters para seguir las convenciones del sprint.

SimpleLinkedList:
Lista enlazada simple con operaciones para agregar al inicio, agregar al final, eliminar y consultar por índice. Usa Node<T> adaptado del PG-04.

DoubleLinkedList:
Lista doblemente enlazada con navegación hacia adelante y atrás. Se usa en AcademicRecordService para almacenar el expediente académico en orden cronológico. Portada de DoublyLinkedList del PG-04, adaptada a DoubleNode<T> con getters/setters.

CircularLinkedList:
Lista circular simple usada para conservar acciones recientes de la sesión. Portada del PG-04 (CircularLinkedList), simplificada al API next()/getFirst() que requiere el sprint.

CircularDoubleLinkedList:
Lista circular doble usada para el historial de vistas. Permite navegar hacia adelante y hacia atrás como en un navegador. Portada del PG-04 (CircularDoublyLinkedList), adaptada a DoubleNode<T> con los métodos next(), previous() y current().

5. Funcionalidad del Sprint 1

El sistema muestra un menú principal con JavaFX. Desde ahí se carga el módulo de expediente académico. El usuario puede agregar cursos usando campos de texto y combo boxes. Las validaciones evitan campos vacíos, notas fuera de rango y letras en campos numéricos. También se puede cargar una demostración con tres cursos y eliminar un curso seleccionado.

Los datos del expediente se muestran en un TableView con código, nombre, periodo, créditos, nota y estado. Además, cada vez que se agrega o elimina información se guarda una representación en JSON.

6. Pruebas

Se creó la clase ListsTest con JUnit 5. Las pruebas verifican el comportamiento de las cuatro listas requeridas: simple, doble, circular simple y circular doble. También se prueba la eliminación por condición en la lista doble enlazada.

7. Conclusión

El Sprint 1 cumple con la configuración Maven, la aplicación JavaFX, el patrón MVC, las listas enlazadas, el módulo de expediente académico, el historial circular, las pruebas JUnit y los artefactos principales solicitados para la revisión del sprint.
